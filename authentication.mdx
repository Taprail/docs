---
title: "Authentication"
description: "Bearer API keys, per-environment, minted from the partner dashboard."
---

Every merchant API request is authenticated with a secret API key in the `Authorization`
header:

```bash
curl https://api.taprail.app/v1/merchant/orders/ORDER_ID \
  -H "Authorization: Bearer tr_test_a1b2c3d4e5f6789012345678901234ab"
```

## Key format

| Prefix | Environment | Works when |
|---|---|---|
| `tr_test_…` | Sandbox | Immediately after minting |
| `tr_live_…` | Live | After Taprail enables live mode for your account |

The environment is **derived from the key**. There is no separate sandbox base URL and no
way to override the environment in a request. A sandbox key can never create a live charge.

## Managing keys

Keys are minted from the [partner dashboard](https://partner.taprail.app) under
**Developer → API Keys**. The **Sandbox** and **Live** tabs each mint keys for their own
environment; live keys stay locked until live mode is enabled.

- The full key is shown **once** at mint time; only a hash is stored. If you lose it, mint a
  new one.
- **Rotation with grace**: **Rotate (24h grace)** mints a new key and schedules existing keys
  to expire after 24 hours, so you can deploy the new key without downtime.
- **Revocation** is immediate: any integration using a revoked key starts receiving `401`s.

<Warning>
  Keys are server-side secrets. Never embed them in web pages, mobile apps, or client-side
  code. The checkout itself needs no key; the order id is the capability.
</Warning>

## Auth failures

| Status | `error` | Meaning |
|---|---|---|
| 401 | `invalid_api_key` | Missing/malformed `Authorization` header, unknown key, or revoked key |
| 403 | `merchant_suspended` | Your merchant account is suspended. Contact support |
| 403 | `live_mode_disabled` | You used a `tr_live_` key before live mode was enabled for your account |

```json
{
  "error": "live_mode_disabled",
  "message": "Live mode isn't enabled for this account yet. Use a tr_test_ key, or complete the go-live checklist."
}
```
