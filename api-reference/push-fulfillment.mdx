---
title: "Push fulfilment update"
api: "POST https://api.taprail.app/v1/merchant/orders/{id}/status"
description: "Show shipping progress on the customer's order timeline in the app."
---

Adds a fulfilment event — shipped, delivered, or a free-form note — to the customer's order
timeline in the Taprail app. It does **not** change the payment status.

<ParamField path="id" type="uuid" required>The order id.</ParamField>

### Body

<ParamField body="status" type="string" required>
  One of `shipped`, `delivered`, `note`.
</ParamField>
<ParamField body="tracking_no" type="string">
  Tracking number — shown with `shipped` / `delivered`.
</ParamField>
<ParamField body="eta" type="string">
  Expected arrival, free-form (e.g. `2026-06-15`).
</ParamField>
<ParamField body="message" type="string">
  A line for the timeline ("Your order left our Lagos warehouse").
</ParamField>

### Response

<ResponseField name="ok" type="boolean">`true` on success.</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://api.taprail.app/v1/merchant/orders/550e8400-.../status \
  -H "Authorization: Bearer tr_test_..." \
  -H "Content-Type: application/json" \
  -d '{
    "status": "shipped",
    "tracking_no": "TRK123456",
    "eta": "2026-06-15",
    "message": "Your order left our Lagos warehouse."
  }'
```
</RequestExample>

<ResponseExample>
```json 200
{ "ok": true }
```
```json 400
{
  "error": "fulfilment_status_invalid",
  "message": "status must be shipped|delivered|note",
  "param": "status"
}
```
</ResponseExample>

<Tip>
  If the customer is in the app when you push an update, their order screen refreshes in
  realtime.
</Tip>
