---
title: "Create order"
api: "POST https://api.taprail.app/v1/merchant/orders"
description: "Create a payment order and get the checkout handles."
---

Creates an order your customer can pay. The response carries the `deep_link` / `qr_payload`
you hand to the checkout, and the fee Taprail will take.

### Headers

<ParamField header="Authorization" type="string" required>
  `Bearer tr_test_…` or `Bearer tr_live_…`
</ParamField>
<ParamField header="Idempotency-Key" type="string" required>
  A unique key for this creation attempt. Retrying with the same key returns the original
  response verbatim — never a duplicate order. Scoped to your merchant.
</ParamField>

### Body

<ParamField body="reference" type="string" required>
  Your own order reference (cart id, invoice number). Unique per merchant per environment —
  a duplicate returns `409`.
</ParamField>
<ParamField body="amount_kobo" type="integer" required>
  Amount the customer pays, in kobo (₦1 = 100 kobo). Must be positive. In sandbox, the last
  two digits select the outcome — see [test amounts](/environments#deterministic-test-amounts).
</ParamField>
<ParamField body="currency" type="string" default="NGN">
  Only `NGN` is supported.
</ParamField>
<ParamField body="items" type="array" default="[]">
  Cart contents, shown to the customer in the app.
  <Expandable title="item fields">
    <ParamField body="name" type="string" required>Item display name.</ParamField>
    <ParamField body="quantity" type="integer" required>Quantity.</ParamField>
    <ParamField body="price_kobo" type="integer" required>Unit price in kobo.</ParamField>
    <ParamField body="image_url" type="string">Optional image URL.</ParamField>
  </Expandable>
</ParamField>
<ParamField body="customer_phone" type="string">
  Customer phone, for your records and receipt matching.
</ParamField>
<ParamField body="customer_email" type="string">
  Customer email, for your records.
</ParamField>
<ParamField body="callback_url" type="string">
  Per-order webhook destination override (`https://` only). Falls back to your
  dashboard-configured endpoint when omitted.
</ParamField>

### Response

<ResponseField name="id" type="uuid">The order id — keep it.</ResponseField>
<ResponseField name="status" type="string">Always `created`.</ResponseField>
<ResponseField name="deep_link" type="string">
  `https://pay.taprail.app/pay?o=<id>` — opens the Taprail app on mobile.
</ResponseField>
<ResponseField name="qr_payload" type="string">
  Same URL — render as a QR for desktop / point-of-sale.
</ResponseField>
<ResponseField name="amount_kobo" type="integer">Echo of the amount.</ResponseField>
<ResponseField name="fee_kobo" type="integer">
  Taprail's fee, locked in at creation. Your net is `amount_kobo - fee_kobo`.
</ResponseField>
<ResponseField name="expires_at" type="string">
  ISO 8601 UTC. Unpaid orders expire 30 minutes after creation.
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://api.taprail.app/v1/merchant/orders \
  -H "Authorization: Bearer tr_test_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: order-1234-attempt-1" \
  -d '{
    "reference": "order-1234",
    "amount_kobo": 750000,
    "items": [
      { "name": "Leather tote", "quantity": 1, "price_kobo": 750000 }
    ],
    "customer_email": "ada@example.com"
  }'
```
```javascript Node.js
const res = await fetch("https://api.taprail.app/v1/merchant/orders", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.TAPRAIL_SECRET_KEY}`,
    "Content-Type": "application/json",
    "Idempotency-Key": "order-1234-attempt-1",
  },
  body: JSON.stringify({
    reference: "order-1234",
    amount_kobo: 750000,
    items: [{ name: "Leather tote", quantity: 1, price_kobo: 750000 }],
    customer_email: "ada@example.com",
  }),
});
const intent = await res.json();
```
</RequestExample>
