**Pay with Taprail** lets your customers pay you from inside the Taprail app. You create an
order from your server and hand the customer a checkout. On desktop that's a modal with a
QR; on mobile it's a deep link straight into the app. When the payment lands, a signed
webhook tells your server.

## How a payment flows

<Steps>
  <Step title="Create an order from your server">
    `POST /v1/merchant/orders` with your API key, an amount, and your own reference. You get
    back an order id, a deep link, and a QR payload.
  </Step>
  <Step title="Hand the customer the checkout">
    On the web, call `Taprail.open(intent)` from the JS SDK. Desktop gets a modal with a QR,
    mobile goes straight into the Taprail app.
  </Step>
  <Step title="The customer approves in the app">
    They see your business name, the amount, and their cart, and authorize with a tap.
  </Step>
  <Step title="You get the result">
    A signed `order.approved` (or `order.declined`) webhook hits your endpoint. If you set a
    return URL, the browser also comes back to your site.
  </Step>
</Steps>

## Base URL

```text
https://api.taprail.app
```

Requests and responses are JSON. Amounts are integers in kobo (₦1 = 100 kobo) and the only
supported currency is `NGN`.

## At a glance

| | |
|---|---|
| Auth | `Authorization: Bearer tr_test_…` / `tr_live_…` API keys |
| Environments | Sandbox and live, picked by the key. Same base URL |
| Idempotency | `Idempotency-Key` header on order creation |
| Result delivery | Signed webhooks (authoritative), plus the SDK promise and an optional browser return URL |
| Order expiry | 30 minutes by default |

<Note>
  API keys live in the [partner dashboard](https://partner.taprail.app). Sandbox keys work
  the moment you sign up; live keys unlock after a short review.
</Note>
